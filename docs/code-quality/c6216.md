# <a name="c6216"></a>C6216

 Diese Warnung gibt an, dass ein boolescher Wert als verwendet wird, wird ein `HRESULT` ohne explizite Umwandlung. Bei booleschen Typen wird Erfolg durch einen Wert ungleich 0 (null) angegeben. Erfolg (`S_OK`) in `HRESULT` wird durch den Wert 0 (null) angezeigt. Die typische Fehlerwert f端r Funktionen, die ein boolesches False zur端ckgeben ist Status "Erfolg" aus, wenn es als getestet wird ein `HRESULT`. Dies kann zu falschen Ergebnissen f端hren.

## <a name="example"></a>Beispiel
 Der folgende Code generiert diese Warnung:

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

HRESULT f( REFGUID riid1, REFGUID riid2 )
{
  // code ...
  return IsEqual(riid1, riid2);
}
```

 So korrigieren Sie die Warnung unter Verwendung des folgenden Codes

```
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

HRESULT f( REFGUID riid1, REFGUID riid2 )
{
  if (IsEqual(riid1, riid2) == TRUE)
  {
    // code ...
    return S_OK;
  }
  else
  {
    // code ...
    return E_FAIL;
  }
}
```

 Bei dieser Warnung entspricht der `SCODE`-Typ `HRESULT`.
