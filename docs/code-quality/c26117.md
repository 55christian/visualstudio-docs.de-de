# <a name="c26117"></a>C26117
Warnung C26117: Das Freigeben der nicht gehaltenen Sperre \<lock > in der Funktion \<func >.

 Die Erzwingung syntaktisch Bereichs bezogener *Sperr-und Sperr* *Freigabe* Paare inC++ C/Programmen wird von der Sprache nicht durchgeführt. Eine Funktion kann einen Sperr seitigen Effekt einführen, indem Sie eine wahrnehmbare Änderung am Parallelitäts Zustand vornimmt. Eine Lock-Wrapper Funktion erhöht z. b. die Anzahl der Sperr Übernahmen oder die Sperrenanzahl für eine angegebene Sperre. Sie können eine Funktion mit einem Nebeneffekt aus einer Sperr-oder Sperr Freigabe mithilfe von `_Acquires_lock_` bzw. `_Releases_lock_` kommentieren. Ohne derartige Anmerkungen wird erwartet, dass eine Funktion keine Sperrenanzahl ändert, nachdem Sie zurückgegeben wurde. Wenn die Anforderungen und Releases nicht ausgeglichen sind, werden Sie als *verwaist*betrachtet. Warnung C26117 wird ausgegeben, wenn eine Funktion, die nicht mit `_Releases_lock_` Kommentiert wurde, eine Sperre freigibt, die Sie nicht enthält, da die Funktion die Sperre besitzen muss, bevor Sie Sie freigibt.

## <a name="example"></a>Beispiel
 Im folgenden Beispiel wird die Warnung C26117 generiert, da die Funktion `ReleaseUnheldLock` eine Sperre freigibt, die Sie nicht notwendigerweise speichert – der Status `flag` ist mehrdeutig – und es gibt keine Anmerkung, die angibt, dass dies der Fall sein soll.

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
        EnterCriticalSection(&p->cs);
    // code ...
    LeaveCriticalSection(&p->cs);
}
```

## <a name="example"></a>Beispiel
 Mit dem folgenden Code wird das Problem behoben, indem sichergestellt wird, dass die freigegebene Sperre auch unter denselben Bedingungen abgerufen wird.

```cpp

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

int flag;

void ReleaseUnheldLock(DATA* p)
{
    if (flag)
    {
        EnterCriticalSection(&p->cs);
        // code ...
        LeaveCriticalSection(&p->cs);
    }
}
```
