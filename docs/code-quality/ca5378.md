# <a name="ca5378-do-not-disable-servicepointmanagersecurityprotocols"></a>CA5378: Deaktivieren Sie ServicePointManagerSecurityProtocols nicht.

|||
|-|-|
|TypeName|Donotabwitch|
|CheckId|CA5378|
|Kategorie|Microsoft.Security|
|Unterbrechende Änderung|Nicht unterbrechend|

## <a name="cause"></a>Ursache

Ein <xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType> Methodenaufrufe legt `Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols` auf `true`fest.

## <a name="rule-description"></a>Regelbeschreibung

Durch `Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols` das `true` Festlegen von auf werden die (WCF)-Transport Layer Security (WCF)-Verbindungen von Windows Communication Framework auf mithilfe von TLS 1,0 beschränkt. Diese Version von TLS wird als veraltet markiert. Weitere Informationen finden Sie unter [Transport Layer Security (TLS) Best Practices with the .NET Framework](/dotnet/framework/network-programming/tls#switchsystemservicemodeldisableusingservicepointmanagersecurityprotocols).

## <a name="how-to-fix-violations"></a>Behandeln von Verstößen

- Wenn Ihre Anwendung .NET Framework v 4.7 oder höher als Zielversion festgelegt ist, <xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType> können Sie entweder den Methoden Befehl entfernen oder den Wert `false`des Schalters auf festlegen.
- Wenn Ihre Anwendung auf .NET Framework v 4.6.2 oder eine frühere Version abzielt und auf .NET Framework v 4.7 oder höher ausgeführt wird, legen Sie `false`den Wert des Schalters auf fest.
- Weitere Informationen finden Sie unter [bewährte Methoden für Transport Layer Security (TLS) mit der .NET Framework](/dotnet/framework/network-programming/tls) für entschärfungen.

## <a name="when-to-suppress-warnings"></a>Wann sollten Warnungen unterdrückt werden?

Sie können diese Warnung unterdrücken, wenn Sie eine Verbindung mit einem Legacy Dienst herstellen müssen, der nicht für die Verwendung sicherer TLS-Konfigurationen aktualisiert werden kann.
