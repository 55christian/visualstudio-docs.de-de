# <a name="ca5378-do-not-disable-servicepointmanagersecurityprotocols"></a>CA5378: Deaktivieren Sie ServicePointManagerSecurityProtocols nicht.

|||
|-|-|
|TypeName|Donotabwitch|
|CheckId|CA5378|
|Kategorie|Microsoft.Security|
|Unterbrechende Änderung|Nicht unterbrechend|

## <a name="cause"></a>Ursache

Ein <xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType>-Methoden aufruflegt `Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols` auf `true` fest.

## <a name="rule-description"></a>Regelbeschreibung

Durch Festlegen von `Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols` auf `true` werden die TLS-Verbindungen Transport Layer Security (Windows Communication Framework) für die Verwendung von TLS 1,0 beschränkt. Diese Version von TLS wird als veraltet markiert. Weitere Informationen finden Sie unter [Transport Layer Security (TLS) Best Practices with the .NET Framework](/dotnet/framework/network-programming/tls#switchsystemservicemodeldisableusingservicepointmanagersecurityprotocols).

## <a name="how-to-fix-violations"></a>Behandeln von Verstößen

- Wenn Ihre Anwendung .NET Framework v 4.7 oder höher als Zielversion festgelegt ist, können Sie entweder den <xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType>-Methodenaufrufe entfernen oder den Wert des Schalters auf `false` festlegen.
- Wenn Ihre Anwendung auf .NET Framework v 4.6.2 oder früher abzielt und auf .NET Framework v 4.7 oder höher ausgeführt wird, legen Sie den Wert des Schalters auf `false` fest.
- Weitere Informationen finden Sie unter [bewährte Methoden für Transport Layer Security (TLS) mit der .NET Framework](/dotnet/framework/network-programming/tls) für entschärfungen.

## <a name="when-to-suppress-warnings"></a>Wann sollten Warnungen unterdrückt werden?

Sie können diese Warnung unterdrücken, wenn Sie eine Verbindung mit einem Legacy Dienst herstellen müssen, der nicht für die Verwendung sicherer TLS-Konfigurationen aktualisiert werden kann.
